# Decision Records

Use this directory for significant product and architecture decisions.

## When to add a decision

Create a record when a choice:

- Is difficult to reverse
- Changes domain terminology
- Changes demo scope
- Introduces major infrastructure
- Changes privacy behavior
- Changes accountability rules
- Changes streak or progress calculations
- Resolves a meaningful disagreement

## File naming

Use:

`NNNN-short-decision-title.md`

Example:

`0001-use-deterministic-partner-responses-for-demo.md`

## Template

```md
# Decision: Title

## Status

Proposed | Accepted | Rejected | Superseded

## Context

What problem or uncertainty required a decision?

## Decision

What was decided?

## Consequences

What becomes easier, harder, required, or impossible?

## Alternatives considered

What other options were evaluated?

## Date

YYYY-MM-DD
```
